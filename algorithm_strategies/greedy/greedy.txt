What are optimization problems?

- A problem is given that has many solutions
= The problem also has a constraint (budget of 1000)
- The problem also has an objective (max storage)
- eg. Problem: What phone should i buy. But i have a budget of $1000. And i want the MAXIMUM storage
- There will only be 1 optimal solution

How to solve optimization problems
- greedy
- dynamic programming
- branch and bound

Greedy algorithm

Algorithm Greedy (A, n)
{
    solution = empty set
    for i = 1 to n {
        x = select (A, n)
        if feasible(solution, x) {
            solution = solution union x
        }
    }
    return solution
}

- feasible() checks the constraint
- select() needs to be designed such that it helps us get the solution based on the objective function

2 paradigms of greedy (
    1. subset (solution is a subset of input)
    2. order (solution is sorted input)

Knapsack (fractional)
=======================

- You have a knapsack with a max weight, as well as a list of objects, each with profit p and weight w
- What is the maximum profit we can get, accounting for the constraint
- We can also take a fraction of an item

Selection method 1:
- Select one by one starting from highest profit
- Each time we increment profit, and subtract weight from max weight until we hit 0 (last one just add fractional profit)

Selection method 2:
- Select one by one starting from lowest weight possible

Selection method 3:
- Calculate the profit per weight for each object
- Select one by one from the highest profit per weight

- After trying out all, we see that selection method 3 gives us the highest profit
- Now we know that for knapsack problem, this is the optimal solution
- Greedy method involves trying out different selection methods to determine which is the best one

MST
=====
- A spanning tree is a sub graph of a graph having all its vertices
- A spanning tree cannot have a cycle
- Number of edges in a spanning tree = number of vertices - 1
- We can only find a spanning tree if a graph is connected
- 1 graph can have multiple spanning trees

Minimum cost spanning tree problem
- Add weights to the graph now, ost is the sum of the weights
- Optimisation problem Constraint: It needs to be a spanning tree ie. no cycle
- Optimisation problem  Objective: We want to find the spanning tree with minimum cost

2 algos to find MST
- prim
- kruskal

Use cases in real life
======================
- Lets say we have a graph where each node is a city, and each edge is the road between cities
- What road should be constructed so that all cities are connected, but total distance is minimum

- Each node is a computer in a network
- How should we link the computers so total cable needed is minimum

Kruskal Algorithm
===================
Steps:
- Select the edges starting from the smallest
- Whenever we add the edge, check if cycle. If cycle exist, skip this edge
- Repeat until number of edges is v - 1

How to check if cycle exists?
- Use disjoint subset

How do we always select the minimum edge?
- Use a heap (priority queue)

Prims
=====
Steps
1. select any vertex as the starting vertex. Then select its min cost edge, and
increment min cost value.
2. from all selected vertex, select the minimum edge connected to them and connect it

Prims and kruskal always give same result

- we use adjacency matrix
- we also use an array to mark if a vertex has already been selected



Dijkstra
========

- From a single vertex, what is the shortest path to all other vertice
- constraint: path
- objective: min distance

algorithm
initial step
- from starting vertice, find out the distane to all other vertice
- if no direct path, put as infinity

repeating step
- from the remaining vertexes, select the shortest distance one ie. smallest d[x]
    - relax all its connected (non selected) vertices
    - relaxation: if d[u] + c(u,v) < d[v]: d[v] = d[u] + c(u,v)
    - d[x] is the distance from starting vertex to vertex x
    - c(u, v) is the cost between 2 vertex
