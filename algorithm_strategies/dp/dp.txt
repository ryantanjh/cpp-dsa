set method:
- create empty set where set contains tuples (profit, weight) of included obj
- initialise obj 1 in set eg. {(5, 4)}
- for each of the remaining objects
    - loop through the prev array: add this objects profits and weigts
        - { (6, 5) } if obj 2 is (1, 1)
        - only add if weight < constraint to save time
    - merge the 2 arrays: arr 1 is you didnt include, arr 2 is you include
        - combined: { (5, 4), (6, 5) }
- time complexity is 2^n

matrix chain multiplication
- problem: To multiply matrix A X B X C X D, there are multiple ways
- eg. AB X CD or ABC X D. Each has different cost. But which is the minimum cost?
- Cost of multiplying a matrix: A is 4 x 2 and B is 2 x 3
    - cost = 4 x 2 x 3 = 24
Algorithm:
Matrix A, B, C, D has index 1 to 4
Initialise a dimension array d with length 5
A: 3x2  B: 2x4  C: 4x2  D: 2x5
d = [3, 2, 4, 2, 5]

function takes in index of matrix
eg. m(1, 4) means multiplying AxBxCxD
m(2, 3) means BxC
int m(int i , int j ) -> returns min cost {
    if i == j: return 0; // cost of multiplying matrix with itself is nothing
    min_cost = []
    for (int k = i; k < j; k++) {
        cost = m(i, k) + m(k + 1, j) + d[i-1] * d[k] * d[j]
        min_cost.append(cost)
    }
    return min(min_cost)
}

m(1, 4)
cost_1 = m(1, 1) + m(2, 4) + 3 * 2 * 4  => A(BCD) which is 3 * 2 * 4 => cost of multiplying A with BCD
this then calls m(2, 4) which follows the same logic

time: O(n^n) since it in each iteration, it also loops n times

with memoizaion: n^2 as we save data in a n by n matrix, and if i,j exists, dont need calculate again
